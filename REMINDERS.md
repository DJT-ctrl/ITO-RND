**Quick rules (manager request):**

1. **Issue first** — no PR for a bug/improvement/deficit without a GitHub issue.
2. **Reference the issue** — every PR uses `Closes #N`.
3. **One PR per issue** — clear explanations; don’t batch unrelated fixes.
4. **Never push to `main`** — branch → PR → review → merge.
