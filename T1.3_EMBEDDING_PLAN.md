**Correction vs. the model named in Erdal's spec/the section below:**
`models/text-embedding-004` does NOT produce 3072-dim vectors — it's fixed at 768.
Verified against the Gemini API embeddings docs:
- `gemini-embedding-001` — default output is 3072-dim (configurable via
  `output_dimensionality`), supports `task_type`, and embeds each string in a
  `contents` list **independently** — required for per-post batching.
- A newer `gemini-embedding-2` model exists but *aggregates* a whole `contents=[...]`
  list into ONE embedding unless each input is wrapped in its own `Content` object —
  wrong shape for "one embedding per post".

**Actual model used:** `models/gemini-embedding-001`, `output_dimensionality=3072`,
`task_type="RETRIEVAL_DOCUMENT"` (posts indexed now, searched/compared later).

**Key discovery:** `NormalizedPost` (processors/schemas.py) has NO `content` field —
it's a feature-only schema (`extra="forbid"`). `run_embeddings.py` re-joins processed
records back to raw post text by `post_id`, reusing `load_raw_posts` from
`processors/run_pipeline.py`.

**Retry logic:** catches `google.genai.errors.APIError`, retries on `code == 429` or
`500 <= code < 600` with exponential backoff (1s/2s/4s), max 3 attempts; anything else
(4xx) raises immediately.

**Output:** `data/embeddings/linkedin_gemini_<timestamp>.npy`, shape `(n, 3072)`.
Posts with `word_count < 10` or blank `content` are skipped (not embedded).

**Usage:**
```
python -m processors.run_embeddings
python -m processors.run_embeddings --processed-file data/processed/linkedin_20260704T205635Z.jsonl
```

## What embeddings are and why T1.3 needs them

### What embeddings do
- Convert text → fixed-size vector (array of floats)
- Semantically similar text gets similar vectors (measured by cosine similarity)
- Example: "I'm hiring a software engineer" and "Looking for a backend developer" have high similarity despite different words

### What embeddings enable (downstream, not T1.3's scope)
- **Semantic similarity search:** "find posts like this one"
- **Clustering:** group posts by topic without manual tags
- **Complement Phase B:** Phase B finds "hashtags correlate with engagement." Embeddings find "this semantic cluster of posts performs well."

### What embeddings do NOT do
- They don't generate text (not generative AI)
- They don't score/rank posts (that's the engagement benchmark from Phase A)
- They don't "understand" in a human sense — just encode meaning as math

### Cost (Gemini text-embedding-004)
- ~$0.025 per 1M input tokens
- 290 posts × ~100 tokens/post = 29k tokens = **~$0.0007** (under a tenth of a penny)
- Scales linearly with dataset size

### Step 2: Create processors/embedder.py

**File structure:** 2 functions, each <80 lines, heavily commented.

#### Function 1: `embed_batch(records, settings)`

```python
def embed_batch(
    records: list[dict[str, Any]], 
    settings: Settings
) -> tuple[np.ndarray, int]:
    """Embed post content using Gemini Embedding 2 (text-embedding-004).
    
    Returns:
        (vectors, skipped_count)
        vectors: shape (n_valid_posts, 3072) — only includes posts with word_count >= 10
        skipped_count: how many posts were skipped due to short/empty content
    
    Raises:
        ValueError if GEMINI_API_KEY is not set in settings
    
    Batching:
        Gemini embeddings API supports batching. We use batch_size=100 to stay 
        well under any rate limits and keep retries cheap if one batch fails.
    
    Retries:
        Exponential backoff on rate limit errors (429) and transient 5xx.
        Max 3 attempts per batch. Fails loudly on 4xx client errors (bad input).
    """
```

**Key logic:**
1. Filter out records where `word_count < 10`, track count
2. Extract `content` field from each valid record
3. Chunk into batches of 100
4. For each batch:
   - Call `client.models.embed_content(model="models/text-embedding-004", contents=batch)`
   - Retry logic: catch rate limit/5xx, sleep with exponential backoff (1s, 2s, 4s), fail after 3 attempts
   - Accumulate embeddings into a list
5. Convert to numpy array shape `(n, 3072)`, return along with skip count

**Gemini API details:**
- Model: `models/text-embedding-004` (Gemini Embedding 2)
- Output dimension: 3072 (matches Erdal's spec exactly)
- Batching: supported by passing a list of strings to `contents=` parameter

#### Function 2: `save_embeddings(vectors, platform, base_dir)`

```python
def save_embeddings(
    vectors: np.ndarray, 
    platform: str, 
    base_dir: str = "data/embeddings"
) -> Path:
    """Persist embedding vectors to a timestamped .npy file.
    
    Output filename: <platform>_gemini_<timestamp>.npy
    Example: linkedin_gemini_20260704T210000Z.npy
    
    Why .npy:
        - Binary format, compact (3072 floats × 290 posts = ~3.5 MB as .npy vs ~35 MB as JSON)
        - Fast to load later (T1.5 will read this for pgvector insertion)
        - numpy.load() reads it in one line
    
    Pairing with processed data:
        The .npy file does NOT contain post_id or content. It's just the vectors.
        To pair vector[i] with the original post, load both:
            records = [json.loads(line) for line in open('data/processed/linkedin_<ts>.jsonl')]
            # Filter to match embedder's skip threshold
            valid_records = [r for r in records if r['word_count'] >= 10]
            vectors = np.load('data/embeddings/linkedin_gemini_<ts>.npy')
        and zip(valid_records, vectors). Indexes match.
    """
```

**Key implementation:**
- `Path(base_dir).mkdir(parents=True, exist_ok=True)`
- `timestamp = datetime.now(timezone.utc).strftime("%Y%m%dT%H%M%SZ")`
- `file_path = Path(base_dir) / f"{platform}_gemini_{timestamp}.npy"`
- `np.save(file_path, vectors)`
- Return `file_path`

**Total file length:** ~120-150 lines with docstrings/comments, broken into 2 clear functions.

### Step 3: Create processors/run_embeddings.py (CLI entry point)

```python
"""CLI entry point for T1.3: embed a processed dataset.

Usage:
    python -m processors.run_embeddings                                     # most recent JSONL
    python -m processors.run_embeddings --input data/processed/linkedin_20260704T205635Z.jsonl
"""

import argparse
import json
from pathlib import Path
from typing import Optional

from config.settings import load_settings
from processors.embedder import embed_batch, save_embeddings


def find_latest_processed_jsonl(processed_dir: str = "data/processed") -> Path:
    """Return the most recent linkedin_*.jsonl file, sorted by timestamp in filename."""
    files = sorted(Path(processed_dir).glob("linkedin_*.jsonl"), reverse=True)
    if not files:
        raise FileNotFoundError(f"No processed JSONL files found in {processed_dir}")
    return files[0]


def run_embeddings(
    input_path: Optional[Path] = None,
    settings = None
):
    """Main function: load JSONL, embed, save."""
    settings = settings or load_settings()
    input_path = input_path or find_latest_processed_jsonl()
    
    with input_path.open(encoding="utf-8") as fh:
        records = [json.loads(line) for line in fh if line.strip()]
    
    print(f"Loaded {len(records)} records from {input_path.name}")
    print(f"Embedding with Gemini Embedding 2 (3072 dims, this may take a minute)...")
    
    vectors, skipped = embed_batch(records, settings)
    
    output_path = save_embeddings(vectors, "linkedin")
    
    print(f"Embedded {vectors.shape[0]} posts (skipped {skipped} short posts)")
    print(f"Wrote {output_path}")
    print(f"Shape: {vectors.shape} — {vectors.shape[0]} posts × 3072 dimensions")


def _parse_args():
    parser = argparse.ArgumentParser(description=__doc__)
    parser.add_argument(
        "--input",
        type=Path,
        help="Path to processed JSONL file. Defaults to most recent data/processed/linkedin_*.jsonl"
    )
    return parser.parse_args()


if __name__ == "__main__":
    args = _parse_args()
    run_embeddings(input_path=args.input)
```

### 3. Spot-check semantic similarity (sanity test)

```python
import json
import numpy as np
from pathlib import Path

# Load the processed JSONL and embeddings
records = [json.loads(line) for line in open('data/processed/linkedin_20260704T205635Z.jsonl')]
vectors = np.load('data/embeddings/linkedin_gemini_<timestamp>.npy')

# Filter to match (embedder skipped word_count < 10, so zip won't line up otherwise)
valid_records = [r for r in records if r['word_count'] >= 10]

# Find two job posts (is_job_post=True)
job_indices = [i for i, r in enumerate(valid_records) if r['is_job_post']]
if len(job_indices) >= 2:
    i, j = job_indices[0], job_indices[1]
    similarity_job = np.dot(vectors[i], vectors[j]) / (np.linalg.norm(vectors[i]) * np.linalg.norm(vectors[j]))
    print(f"Similarity between two job posts: {similarity_job:.3f}")

# Find a job post and a non-job post
if job_indices:
    non_job_indices = [i for i, r in enumerate(valid_records) if not r['is_job_post']]
    if non_job_indices:
        i, j = job_indices[0], non_job_indices[0]
        similarity_mixed = np.dot(vectors[i], vectors[j]) / (np.linalg.norm(vectors[i]) * np.linalg.norm(vectors[j]))
        print(f"Similarity between job post and non-job post: {similarity_mixed:.3f}")
```

## Nuances / edge cases

### 1. Empty or very short content
- Filter on `word_count < 10` (already computed in Phase A). Log how many were skipped.
- Why 10: Below this, posts are usually just "Congrats!" or "👍" — not semantically meaningful.

### 2. Retry logic
- **Rate limits (429):** Exponential backoff: 1s, 2s, 4s. Max 3 attempts.
- **Transient 5xx:** Same retry pattern.
- **4xx client errors (except 429):** Fail loudly, log which batch caused it.

### 3. Token limits
- Gemini `text-embedding-004` has a per-text limit (~20k tokens/request per individual text).
- LinkedIn posts are short (~50-200 words = 65-260 tokens). No post in the current 290-post dataset is close to 20k tokens.
- Defensive: If a post's `char_count > 80000`, skip it and log a warning.

### 4. Pairing embeddings with records later
- The `.npy` file is JUST vectors, no metadata.
- To pair vector[i] with post_id/content:
  1. Load the SAME JSONL that was input to `run_embeddings.py`
  2. Filter to `word_count >= 10` (same filter the embedder used)
  3. `zip(filtered_records, vectors)` — indexes now match

### 5. Gemini API call pattern for batching
- `embed_content()` takes a `contents` parameter that can be a single string OR a list of strings for batch
- For simplicity, send one string at a time in a loop, batching the API calls (not Gemini's internal batching)

### 6. Why 3072 dimensions (Gemini Embedding 2)
- **This implementation uses 3072** (Gemini Embedding 2's fixed output).
- **Matches Erdal's spec exactly** (3072 dims requested).
- **Terminal output should say "3072 dimensions (Gemini Embedding 2)"** so it's obvious.
